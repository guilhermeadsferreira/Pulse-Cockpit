# Phase 999.5: Gemini Preprocessing Pass Summary

**One-liner:** Implementa "Pass 0" de pré-processamento usando Gemini Flash para limpar transcrições antes de enviar ao Claude, reduzindo consumo de tokens em ~60%.

## Problema Resolvido

Três 1:1s consumiam ~35% do consumo do Claude Code CLI do período. A transcrição bruta de voz contém muito ruído ("né", "tipo", repetições, frases incompletas) que é processado por todos os passes subsequentes:

- Pass 1 (identificação)
- Pass 2 (com perfil.md)
- 1:1 Deep Pass (o maior consumidor)

## Solução

**Pipeline novo:**
```
Transcript bruto (10–30K chars)
    ↓ Pass 0: Gemini Flash (Google AI API)
Transcript limpo/estruturado (2–5K chars)
    ↓ Pass 1 → Pass 2 → 1:1 Deep (Claude)
```

## Comportamento de Falha (CRÍTICO)

| Situação | Resultado |
|----------|-----------|
| **Pré-processamento DESATIVADO** | ✅ Pipeline continua normalmente com texto bruto |
| **Pré-processamento ATIVADO e FUNCIONA** | ✅ Usa texto limpo, economiza ~60% tokens |
| **Pré-processamento ATIVADO mas FALHA** | 🛑 **PARA O PIPELINE** com erro explicativo |

**Por que falha hard?** Se o usuário ativou o pré-processamento explícito para economizar tokens, um fallback silencioso para texto bruto resultaria em consumo máximo do Claude sem aviso. É melhor parar e dar ao usuário a chance de:
1. Adicionar créditos ao Google AI Studio, **ou**
2. Desativar o pré-processamento nas Settings

**Mensagem de erro:**
```
Pré-processamento Gemini falhou: {erro}. 
Verifique seus créditos em aistudio.google.com 
ou desative o pré-processamento nas Settings.
```

## O que foi implementado

### 1. Prompt de Pré-processamento (`gemini-preprocessing.prompt.ts`)

Prompt determinístico que:
- **Remove:** preenchedores ("né", "tipo", "entendeu"), interjeições, sons transcritos, repetições, frases incompletas sem valor
- **Preserva:** nomes, datas, decisões, ações, feedbacks, bloqueios, métricas, termos técnicos
- **Estrutura:** participantes → contexto → pontos discutidos → decisões → ações → observações

### 2. Módulo GeminiPreprocessor (`GeminiPreprocessor.ts`)

- Chamada REST à Google AI API (`generativelanguage.googleapis.com`)
- Modelo: `gemini-1.5-flash`
- Temperatura: 0.1 (determinístico)
- Timeout: 15s
- Limite de input: 100K caracteres (truncamento seguro)
- Retorna sucesso/falha com mensagem de erro (pipeline decide o que fazer)

### 3. Integração no Pipeline (`IngestionPipeline.ts`)

```typescript
// No início de processItem()
const geminiActive = !!(settings.useGeminiPreprocessing && settings.googleAiApiKey)
if (geminiActive) {
  const result = await preprocessTranscript(...)
  if (result.success) {
    text = result.cleanedText
    log: "Transcript reduzido de X para Y chars (Z% economia)"
  } else {
    // FALHA HARD: se ativado mas falhou, paramos
    throw new Error(`Pré-processamento Gemini falhou: ${result.error}...`)
  }
}
```

### 4. UI de Configuração (`SettingsView.tsx`)

Nova seção "Pré-processamento Gemini":
- Campo API Key (type=password, link para aistudio.google.com)
- Toggle "Usar Gemini Flash para pré-processar transcrições"
  - Disabled quando API key ausente
  - Hint dinâmico com explicação do benefício

### 5. Settings (`SettingsManager.ts` + `ipc.ts`)

Novos campos em `AppSettings`:
- `googleAiApiKey?: string` — API key do Google AI
- `useGeminiPreprocessing?: boolean` — toggle de ativação

## Economia de Tokens

| Cenário | Input Pass 1 | Input Pass 2 | Input 1:1 Deep | Total |
|---------|--------------|--------------|----------------|-------|
| **Sem pré-processamento** | 10-30K chars | 10-30K + perfil | 10-30K + contexto | Alto |
| **Com pré-processamento** | 2-5K chars | 2-5K + perfil | 2-5K + contexto | ~60% menor |

**Estimativa:** Redução de 60-70% no consumo de tokens do Claude para transcrições de voz.

## Como Usar

1. Obter API key em [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)
2. Abrir Settings no Pulse Cockpit
3. Colar a key no campo "Google AI API Key"
4. Ativar o toggle "Usar Gemini Flash para pré-processar transcrições"
5. Nova ingestão usará automaticamente o pré-processamento

## Design Decisions

| Decisão | Alternativa | Por que escolhemos isso |
|---------|-------------|------------------------|
| Google AI API direta | Via OpenRouter | Mais controle, preço previsível, sem rate limits de terceiros |
| Gemini Flash | Gemini Pro | Suficiente para limpeza de texto, 10x mais barato |
| **Falha hard** | Fallback silencioso para texto bruto | Se usuário ativou para economizar, fallback silencioso gasta tokens Claude sem aviso — é melhor parar e informar |
| Temperatura 0.1 | 0.0 ou 0.7 | Determinístico mas com pequena flexibilidade para frases naturais |
| Timeout 15s | 30s ou 60s | Não bloquear pipeline indefinidamente se Gemini estiver lento |
