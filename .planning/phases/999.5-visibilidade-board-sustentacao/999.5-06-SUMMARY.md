## Decisions Made

- `fetchAndCacheSustentacao` reutilizada no handler — mesma função do handler `sustentacao:refresh`, garante cache consistente sem chamar Jira duas vezes
- `SIGNIFICANT_TICKET_THRESHOLD = 3` como constante nomeada — facilita ajuste futuro
- Graceful degradation por assignee individual: se um falha ao persistir, os demais continuam e o resultado retorna normalmente ao frontend
- Timeout 90_000ms para análise via Claude CLI — consistente com outros handlers de análise longa no projeto
